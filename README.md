 Análisis de Campaña de Phishing: Infraestructura AWS & SmishingEste repositorio contiene el análisis técnico detallado de una campaña activa de Smishing (phishing vía SMS) que utiliza infraestructura en la nube de Amazon Web Services (AWS S3 y CloudFront) para evadir detecciones y distribuir contenido malicioso.📊 Resumen de HallazgosSe identificó una cadena de ataque que utiliza ingeniería social basada en urgencia financiera para inducir a las víctimas a interactuar con un enlace acortado, el cual redirige a un sitio de phishing alojado en una infraestructura estática de AWS.🔗 Cadena de AtaqueVector Inicial: Mensaje SMS prometiendo un abono de 300.000 COP$.Evasión: Uso de URL acortada (did.li) para ocultar el destino final.Backend: Hosting estático en Amazon S3.Entrega: Red de distribución de contenido (CDN) CloudFront para mejorar la disponibilidad y evadir bloqueos por IP.🛠️ Detalles Técnicos de la InfraestructuraComponenteTecnología DetectadaHallazgos ClaveHostingAWS S3Configuración de hosting estático.CDNAWS CloudFrontPresencia de caché (Hit from cloudfront).RedirecciónHTTP 301Salto automático de did.li hacia xy321.club.SeguridadWAF / Rate LimitRespuestas HTTP 429 ante escaneos intensivos.🔍 Análisis de Vulnerabilidades y Exposición1. Enumeración de Contenido (Gobuster)Durante el análisis de rutas, se detectaron recursos sensibles protegidos por el servidor:Acceso Denegado (403): .git/HEAD, .ssh, .bash_history, .htpasswd.Rate Limiting (429): Activado tras múltiples peticiones a index.html y robots.txt.2. Errores de ConfiguraciónSe identificó un error 404 NoSuchKey en ciertas rutas, lo que revela detalles internos de la infraestructura como el RequestId y el HostId de AWS, útiles para tareas de depuración o reconocimiento técnico.3. Evidencia de Reconocimiento (DIRB)El escaneo con herramientas de fuerza bruta reveló rutas activas con código de estado 200 OK, confirmando la presencia de directorios operativos como /soap.🚨 Conclusiones de SeguridadSeveridad: ALTA. El ataque está diseñado para el robo de credenciales y fraude financiero masivo.Técnicas: Uso sofisticado de infraestructura cloud para dificultar el "takedown" del sitio.Recomendación: Bloqueo de los dominios identificados y educación al usuario final sobre mensajes de texto sospechosos.
